# Player Template — canonical schemas (for /awaken inventory & repair)

Mirror of the Seed v1.0.0 Player Template. Used to verify an instance and rebuild missing pieces. Notion create-database DDL syntax. **Dependency order matters** — relations reference earlier databases (use the actual data source IDs from the player's workspace, discovered or just created).

## Pages (skeletons)

- **The System — Job Search HQ** (root, ⚔️) — [SYSTEM] welcome callout · "How this works (30-second version)" · Sections link list.
- **📖 Operating Manual** — player-edition manual (LLM-wiki principle · Agent Protocol · Command Codex (22) · Game Rules · The Nexus & patches · SLA engine · Weekly rhythm · Conventions).
- **🪟 Status Window** — Player Card callout (Level 0 "Unawakened" until /awaken) · sections: Positioning / Experience Ledger / Metrics Bank / Education & Extras / CV & LinkedIn Versions / Interview Q&A Prep · child dbs: Story Bank, Competency Matrix, XP Ledger, Achievements.
- **🏰 Guild Hall** — networking flow + follow-up ladder + Parties note · child db: Networking Events.
- **🩹 Patch Notes** — how patches/corrections work · child db: Questions & Feedback Log.
- **🎭 Theme Registry** — skin-not-skeleton callout + 3-theme name mapping + switch procedure.
- **🧬 Kernel** — Instance ID table · Player · Versions · Nexus links · Sharing toggles · Party links · Sealed Ledger.

## Databases (dependency order)

1. **Gate Intel**: "Company" TITLE, "Industry" RICH_TEXT, "Website" URL, "HQ / Locations" RICH_TEXT, "Research Status" SELECT('Not Started','In Progress','Complete','Needs Refresh'), "Last Refreshed" DATE, "Why This Company" RICH_TEXT, "Glassdoor / Culture Notes" RICH_TEXT
2. **Hunter Network**: "Name" TITLE, "Type" SELECT('Recruiter','Hiring Manager','Interviewer','Referral','Network','Other'), "Warmth" SELECT('Hot','Warm','Cold'), "Company" RICH_TEXT, "Companies" RELATION(→Gate Intel, DUAL 'Key People'), "Role / Title" RICH_TEXT, "Email" EMAIL, "LinkedIn" URL, "Notes" RICH_TEXT, "Conversation Notes" RICH_TEXT, "Research Brief" SELECT('Not Researched','Researched — see page','Not Findable Online'), "Follow-up Ladder" SELECT('T+2d Thank You','T+3w Value Touch','T+7w Relationship Touch','Ongoing','None'), "Last Contacted" DATE, "Next Follow-up" DATE, "Follow-up Status" FORMULA('if(empty(prop("Next Follow-up")), "—", if(prop("Next Follow-up") <= now(), "🔔 Due", "🟢 Scheduled"))')
3. **Quest Board**: "Role" TITLE, "Job ID" UNIQUE_ID PREFIX 'QST', "Stage" SELECT('Saved','Applied','Recruiter Screen','Hiring Manager','Panel / Case','Final Round','Offer','Rejected','Closed – No Response','Withdrawn'), "Priority" SELECT('🔥 High','Medium','Low'), "Source" SELECT('LinkedIn','Seek','Referral','Recruiter Outreach','Company Site','Other'), "Company" RELATION(→Gate Intel, DUAL 'Open Roles'), "Contacts" RELATION(→Hunter Network, DUAL 'Roles'), "Location" RICH_TEXT, "Salary Range" RICH_TEXT, "Job Link" URL, "CV Version Sent" RICH_TEXT, "Next Action" RICH_TEXT, "Next Action Due" DATE, "Applied On" DATE, "Last Activity" DATE, "Fit Score" NUMBER, "Deep Research Done" CHECKBOX, "Talking Points Ready" CHECKBOX, "Days Silent" FORMULA('if(empty(prop("Last Activity")), toNumber(""), dateBetween(now(), prop("Last Activity"), "days"))'), "SLA Status" FORMULA('if(or(or(prop("Stage") == "Offer", prop("Stage") == "Rejected"), or(prop("Stage") == "Withdrawn", prop("Stage") == "Closed – No Response")), "—", if(empty(prop("Last Activity")), "—", if(dateBetween(now(), prop("Last Activity"), "days") >= 14, "🔴 Fading Gate", if(dateBetween(now(), prop("Last Activity"), "days") >= 2, "🟠 Hesitation", "🟢 On track"))))')
   — Note: "Closed – No Response" uses an en dash. Single-char UNIQUE_ID prefixes are rejected; use 'QST'. Deep 4-arg or() fails; nest 2-arg or()s as shown.
4. **Battle Log**: "Subject / Summary" TITLE, "Date" DATE, "Direction" SELECT('Sent','Received'), "Channel" SELECT('Email','LinkedIn','Phone / Call','In-person','Application Portal','Other'), "Message Type" SELECT('Application','Follow-up','Thank You','Networking / Intro','Negotiation','Scheduling','Feedback','Other'), "Person" RELATION(→Hunter Network, DUAL 'Messages'), "Related Role" RELATION(→Quest Board, DUAL 'Comms Log'), "Outcome" SELECT('Positive','Neutral','Negative','No Reply Yet'), "Response Received" CHECKBOX, "Response Summary" RICH_TEXT — full message text lives in each entry's page body.
5. **Story Bank** (child of Status Window): "Story" TITLE, "Competencies" MULTI_SELECT('Leadership','Strategy','Execution / Delivery','Stakeholder Mgmt','Commercial / P&L','Innovation','Resilience / Conflict','People Development','Data / Analytics'), "Strength" SELECT('Signature story','Solid','Needs work'), "Metric Inside" RICH_TEXT, "Used In" RELATION(→Quest Board, DUAL 'Stories Used'), "Last Rehearsed" DATE
6. **Competency Matrix** (child of Status Window): "Competency" TITLE, "Category" SELECT('Leadership','Strategic','Commercial','Technical / Craft','Communication / Brand'), "Self Rating (1-5)" NUMBER, "Claude Rating (1-5)" NUMBER, "Evidence Strength" SELECT('Strong — proven with metrics','Moderate — thin evidence','Gap — needs building'), "Talking Point Angle" RICH_TEXT, "Benchmark Notes" RICH_TEXT, "Development Actions" RICH_TEXT
7. **XP Ledger** (child of Status Window): "Action" TITLE, "Date" DATE, "XP" NUMBER, "Category" SELECT('Application','Interview','Networking','Follow-up','Learning','Achievement Bonus','Weekly Ritual','Streak Bonus','Bonus Quest','Penalty','Physical','Mental','Hobby','Awakening','Other'), "Related Quest" RELATION(→Quest Board, DUAL 'XP Earned'), "Notes" RICH_TEXT
8. **🗒 Agent Notes**: "Note" TITLE, "Date" DATE, "Type" SELECT('Appraisal','Scout','Raid Prep','Interview Prep','Cover Letter','Correction','Other'), "Related Quest" RELATION(→Quest Board, DUAL 'Agent Notes'), "Status" SELECT('Current','Superseded'), "Summary" RICH_TEXT
9. **Daily Log**: "Day" TITLE, "Date" DATE, "XP Earned Today" NUMBER, "Streak After Today" NUMBER, "Quests Touched" RELATION(→Quest Board, DUAL 'Daily Log'), "Domains Touched" MULTI_SELECT('Career','Physical','Mental','Hobby'), "Player Mood / Energy" SELECT('🔥 Charged','Steady','Grinding','Low','Not logged'), "Debuffs Cleared" RICH_TEXT, "Bonus Quest Active" CHECKBOX, "System Notes" RICH_TEXT, "Welcome # Delivered" NUMBER
10. **Networking Events** (child of Guild Hall): "Event" TITLE, "Date" DATE, "Type" SELECT('Meetup','Conference','Industry Dinner','Online / Webinar','Alumni','Recruiter Event','Other'), "Status" SELECT('Upcoming','Attended','Skipped'), "Location" RICH_TEXT, "People Met" RELATION(→Hunter Network, DUAL 'Met At'), "Debrief Done" CHECKBOX, "Worth Repeating" SELECT('Yes','Maybe','No')
11. **Achievements** (child of Status Window): "Achievement" TITLE, "Kind" SELECT('Achievement','Badge'), "Track" SELECT('Gatecrasher','Relentless','Guildmaster','Boss Hunter','Chronicler','Unbroken','Special'), "Badge Tier" SELECT('I','II','III','IV','V','Mythic'), "Rarity" SELECT('Common','Rare','Epic','Legendary'), "Status" SELECT('🔒 Locked','🏆 Unlocked'), "Unlock Criteria" RICH_TEXT, "Unlocked On" DATE, "XP Bonus" NUMBER — ships seeded: 12 Kind=Achievement rows + 6 Tier I badge rows, all 🔒 Locked. If missing and the template can't be re-duplicated, recreate the database empty and log an Open Question for the Game Admin to resend the seed rows.
12. **Questions & Feedback Log** (child of Patch Notes): "Entry" TITLE, "Date" DATE, "Type" SELECT('Open Question','Answered Question','Correction','Not Working','Enhancement Suggestion','Decision','Session Note'), "Status" SELECT('Open','Answered','Adopted','Declined','Actioned','Standing Rule'), "Area" SELECT('Workspace Design','Search Strategy','CV / Materials','Networking','Interviewing','Agent Behaviour','Other'), "Answer / Resolution" RICH_TEXT, "Raised By" SELECT('Player','Agent')

Select option colors: match the template where possible; color drift is cosmetic and not worth a repair cycle.
